## What is coc.nvim?

coc.nvim enhances your Vim or Neovim to match the user experience provided by VS Code through a rich extension ecosystem and a full implementation of the [Language Server Protocol (LSP 3.17)](https://microsoft.github.io/language-server-protocol/).

It runs in a **separate NodeJS process**, keeping your editor fast and responsive. This process hosts language servers and VS Code-compatible extensions, communicating with Vim/Neovim over a typed RPC interface.

<Note>
  coc.nvim does not include built-in support for any specific language. You
  must install coc extensions or configure language servers separately to
  enable LSP features.
</Note>

## How it works

coc.nvim starts a long-lived NodeJS child process when Vim launches. That process:

- Loads **VS Code-compatible extensions** from npm or from your `runtimepath`
- Spawns and manages **language server processes** on your behalf
- Implements the full LSP client, forwarding results back to Vim as completions, diagnostics, hover docs, and more
- Exposes a **JSON configuration file** (`coc-settings.json`) that works exactly like VS Code's `settings.json`

Because the heavy lifting happens outside Vim's main loop, most operations are non-blocking and will not slow down your editing experience.

## Requirements

<Note>
  **Vim** >= 9.0.0438 or **Neovim** >= 0.8.0 is required.
  **Node.js** >= 16.18.0 must be available on your `$PATH`.
</Note>

| Requirement | Minimum version |
|-------------|-----------------|
| Vim         | 9.0.0438        |
| Neovim      | 0.8.0           |
| Node.js     | 16.18.0         |

For Neovim users, run `:checkhealth` after installation to verify that all requirements are satisfied.
