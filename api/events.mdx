---
title: "events"
description: "The events module provides a typed pub/sub system for Vim/Neovim editor events such as cursor moves, buffer changes, insert mode transitions, and more."
---

The `events` module is a singleton that bridges Vim/Neovim autocmd events into JavaScript handlers. Access it via:

```javascript
const { events } = require('coc.nvim')
```

---

## Subscribing to events

### events.on

```typescript
events.on(
  event: AllEvents | AllEvents[],
  handler: (...args: any[]) => void | Promise<void>,
  thisArg?: any,
  disposables?: Disposable[] | true
): Disposable
```

Registers a handler for one or more events. Returns a `Disposable` that removes the handler when disposed.

- Pass an array of event names to subscribe to multiple events with a single handler.
- Pass `context.subscriptions` (a `Disposable[]`) as `disposables` to auto-unsubscribe on deactivation.
- Pass `true` as `disposables` to make it a one-time handler (fires once, then unsubscribes).

```javascript
// Single event
context.subscriptions.push(
  events.on('BufEnter', (bufnr) => {
    console.log(`Entered buffer: ${bufnr}`)
  })
)

// Multiple events with shared handler
events.on(['InsertEnter', 'InsertLeave'], (bufnr) => {
  console.log(`Insert mode changed in buffer: ${bufnr}`)
}, null, context.subscriptions)
```

---

### events.once

```typescript
events.once(
  event: AllEvents,
  handler: (...args: any[]) => void | Promise<void>,
  thisArg?: any
): Disposable
```

Like `events.on`, but the handler is invoked at most once then automatically removed. Equivalent to passing `true` as `disposables`.

```javascript
events.once('ready', () => {
  console.log('coc.nvim is ready')
})
```

---

### events.race

```typescript
events.race(
  events: AllEvents[],
  token?: number | CancellationToken
): Promise<{ name: AllEvents; args: unknown[] } | undefined>
```

Returns a `Promise` that resolves with the first event from `events` that fires. Useful for waiting on user action.

- Pass a `number` as `token` to set a timeout in milliseconds â€” resolves `undefined` on timeout.
- Pass a `CancellationToken` as `token` to cancel waiting externally â€” resolves `undefined` on cancellation.

```javascript
// Wait up to 5 seconds for the user to enter or leave insert mode
const result = await events.race(['InsertEnter', 'InsertLeave'], 5000)
if (result) {
  console.log(`Got event: ${result.name}`, result.args)
} else {
  console.log('Timed out')
}
```

---

### events.fire

```typescript
events.fire(event: string, args: any[]): Promise<void>
```

Manually fires an event with the given arguments. Primarily used in tests.

```javascript
await events.fire('BufEnter', [1])
```

## State properties

These read-only properties reflect the current editor state tracked by the events system.

<ResponseField name="insertMode" type="boolean">
  `true` when Vim is in insert mode (mode starts with `'i'` and `_insertMode` flag is set).
</ResponseField>

<ResponseField name="pumvisible" type="boolean">
  `true` when the completion popup menu is visible.
</ResponseField>

<ResponseField name="pumAlignTop" type="boolean">
  `true` when the popup menu is aligned to the top of the screen.
</ResponseField>

<ResponseField name="pumInserted" type="boolean">
  `true` when the popup menu inserted a completion item.
</ResponseField>

<ResponseField name="completing" type="boolean">
  `true` while a completion session is active.
</ResponseField>

<ResponseField name="cursor" type="CursorPosition">
  The most recently tracked cursor position.

  ```typescript
  interface CursorPosition {
    readonly bufnr: number
    readonly lnum: number   // 1-based
    readonly col: number    // 1-based byte column
    readonly insert: boolean
  }
  ```
</ResponseField>

<ResponseField name="bufnr" type="number">
  The buffer number of the most recently active buffer.
</ResponseField>

<ResponseField name="mode" type="string">
  The current Vim mode string (e.g., `'n'`, `'i'`, `'v'`).
</ResponseField>

<ResponseField name="ready" type="boolean">
  `true` once the `'ready'` event has been fired.
</ResponseField>

<ResponseField name="lastChangeTs" type="number">
  Unix timestamp (ms) of the most recent `TextChanged`, `TextChangedI`, or `TextChangedP` event.
</ResponseField>

<ResponseField name="requesting" type="boolean">
  `true` while coc.nvim is performing a synchronous request to Vim. Slow event handlers are logged when this is `true`.
</ResponseField>

<ResponseField name="timeout" type="number">
  Threshold in milliseconds after which a slow handler warning is logged. Default: `1000`.
</ResponseField>

### Text change events

<Tabs>
  <Tab title="TextChanged">
    ```
    handler: (bufnr: number, changedtick: number) => void
    ```
    Fires when a buffer's text changes in normal mode.
  </Tab>
  <Tab title="TextChangedI">
    ```
    handler: (bufnr: number, info: InsertChange) => void
    ```
    Fires when a buffer's text changes in insert mode.

    ```typescript
    interface InsertChange {
      readonly lnum: number
      readonly col: number
      readonly line: string
      readonly changedtick: number
      pre: string             // text before cursor
      insertChar?: string     // character that triggered this change
      insertChars?: string[]  // recent inserted characters
    }
    ```
  </Tab>
  <Tab title="TextChangedP">
    ```
    handler: (bufnr: number, info: InsertChange) => void
    ```
    Fires when text changes while the popup menu is visible.
  </Tab>
  <Tab title="TextInsert">
    ```
    handler: (bufnr: number, info: InsertChange, character: string) => void
    ```
    Fires (via `process.nextTick`) after `TextChangedI` when a recognized character was inserted.
  </Tab>
</Tabs>

## Usage examples

<CodeGroup>
```javascript Cursor hold action
const { events, workspace } = require('coc.nvim')

events.on('CursorHold', async (bufnr) => {
  const doc = workspace.getDocument(bufnr)
  if (!doc) return
  // trigger diagnostic refresh, etc.
}, null, context.subscriptions)
```

```javascript One-shot ready handler
const { events, window } = require('coc.nvim')

events.once('ready', () => {
  window.showMessage('coc.nvim is ready!')
})
```

```javascript Race between events
const { events } = require('coc.nvim')

async function waitForUserAction() {
  const result = await events.race(
    ['InsertEnter', 'CursorMoved', 'BufEnter'],
    3000  // 3 second timeout
  )
  if (!result) return 'timeout'
  return result.name
}
```

```javascript Insert mode tracking
const { events } = require('coc.nvim')

events.on('TextChangedI', (bufnr, info) => {
  if (info.insertChar === '.') {
    // trigger member completion
  }
}, null, context.subscriptions)
```
</CodeGroup>
