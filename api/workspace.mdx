---
title: "workspace"
description: "The workspace module provides access to open documents, configuration, file system operations, and workspace folder management."
---

The `workspace` module is the primary interface for interacting with the editor environment. Access it via:

```javascript
const { workspace } = require('coc.nvim')
```

## Events

All events return an `Event<T>` which can be subscribed to via `.on(listener)` or `.on(listener, disposables)`.

<AccordionGroup>
  <Accordion title="Document lifecycle events">
    <ResponseField name="onDidOpenTextDocument" type="Event<LinesTextDocument>">
      Fires when a new text document is opened and attached.
    </ResponseField>
    <ResponseField name="onDidCloseTextDocument" type="Event<LinesTextDocument>">
      Fires when a text document is closed.
    </ResponseField>
    <ResponseField name="onDidChangeTextDocument" type="Event<DidChangeTextDocumentParams>">
      Fires when the content of an open text document changes.
    </ResponseField>
    <ResponseField name="onDidSaveTextDocument" type="Event<LinesTextDocument>">
      Fires after a document is saved.
    </ResponseField>
    <ResponseField name="onWillSaveTextDocument" type="Event<TextDocumentWillSaveEvent>">
      Fires before a document is saved. Handlers may provide edits to apply before saving.
    </ResponseField>
  </Accordion>

  <Accordion title="Configuration & folder events">
    <ResponseField name="onDidChangeConfiguration" type="Event<IConfigurationChangeEvent>">
      Fires when the user's configuration changes.
    </ResponseField>
    <ResponseField name="onDidChangeWorkspaceFolders" type="Event<WorkspaceFoldersChangeEvent>">
      Fires when workspace folders are added or removed.
    </ResponseField>
    <ResponseField name="onDidRuntimePathChange" type="Event<string[]>">
      Fires when new directories are added to Vim's `runtimepath`.
    </ResponseField>
  </Accordion>

  <Accordion title="File operation events">
    <ResponseField name="onDidCreateFiles" type="Event<FileCreateEvent>">
      Fires after files are created via `workspace.createFile()`.
    </ResponseField>
    <ResponseField name="onDidRenameFiles" type="Event<FileRenameEvent>">
      Fires after files are renamed via `workspace.renameFile()`.
    </ResponseField>
    <ResponseField name="onDidDeleteFiles" type="Event<FileDeleteEvent>">
      Fires after files are deleted via `workspace.deleteFile()`.
    </ResponseField>
    <ResponseField name="onWillCreateFiles" type="Event<FileWillCreateEvent>">
      Fires before files are created. Handlers may return a `WorkspaceEdit`.
    </ResponseField>
    <ResponseField name="onWillRenameFiles" type="Event<FileWillRenameEvent>">
      Fires before files are renamed. Handlers may return a `WorkspaceEdit`.
    </ResponseField>
    <ResponseField name="onWillDeleteFiles" type="Event<FileWillDeleteEvent>">
      Fires before files are deleted. Handlers may return a `WorkspaceEdit`.
    </ResponseField>
  </Accordion>
</AccordionGroup>

## Methods

### getConfiguration

```typescript
getConfiguration(section?: string, scope?: ConfigurationResourceScope): WorkspaceConfiguration
```

Returns the configuration scoped to an optional `section` and optional resource URI or language ID. The returned `WorkspaceConfiguration` supports `.get()`, `.has()`, `.inspect()`, and `.update()`.

<CodeGroup>
```javascript Basic usage
const config = workspace.getConfiguration('coc.preferences')
const timeout = config.get<number>('timeout', 500)
```

```javascript Scoped to a resource
const doc = await workspace.document
const config = workspace.getConfiguration('editor', doc.uri)
const tabSize = config.get<number>('tabSize', 2)
```
</CodeGroup>

---

### getDocument

```typescript
getDocument(uri: number | string): Document | null | undefined
```

Returns the coc `Document` for the given buffer number or URI. Returns `null` or `undefined` when the buffer does not exist or is not attached.

```javascript
const doc = workspace.getDocument(workspace.bufnr)
if (doc) {
  console.log(doc.filetype)
}
```

---

### getAttachedDocument

```typescript
getAttachedDocument(uri: number | string): Document
```

Like `getDocument`, but throws if the document does not exist or is not attached.

---

### openTextDocument

```typescript
openTextDocument(uri: URI | string): Promise<Document>
```

Opens a file URI as a coc document, loading it into a buffer if not already open.

```javascript
const doc = await workspace.openTextDocument('/path/to/file.ts')
```

---

### applyEdit

```typescript
applyEdit(edit: WorkspaceEdit): Promise<boolean>
```

Applies an LSP `WorkspaceEdit` to the workspace. Returns `true` on success.

```javascript
const success = await workspace.applyEdit(workspaceEdit)
```

---

### getWorkspaceFolder

```typescript
getWorkspaceFolder(uri: string | URI): WorkspaceFolder | undefined
```

Returns the workspace folder that contains the given URI, or `undefined` if none matches.

```javascript
const folder = workspace.getWorkspaceFolder(doc.uri)
if (folder) {
  console.log(folder.uri)
}
```

---

### findFiles

```typescript
findFiles(
  include: GlobPattern,
  exclude?: GlobPattern | null,
  maxResults?: number,
  token?: CancellationToken
): Promise<URI[]>
```

Finds files in the workspace matching a glob pattern.

```javascript
const uris = await workspace.findFiles('**/*.ts', '**/node_modules/**', 100)
for (const uri of uris) {
  console.log(uri.fsPath)
}
```

### createFileSystemWatcher

```typescript
createFileSystemWatcher(
  globPattern: GlobPattern,
  ignoreCreate?: boolean,
  ignoreChange?: boolean,
  ignoreDelete?: boolean
): FileSystemWatcher
```

Creates a file system watcher. Does not fail when watchman is not installed — it degrades gracefully.

```javascript
const watcher = workspace.createFileSystemWatcher('**/*.json')
watcher.onDidChange(uri => {
  console.log('changed:', uri.fsPath)
})
context.subscriptions.push(watcher)
```

---

### registerTextDocumentContentProvider

```typescript
registerTextDocumentContentProvider(
  scheme: string,
  provider: TextDocumentContentProvider
): Disposable
```

Registers a provider that supplies content for URIs with a custom scheme. The provider must implement `provideTextDocumentContent(uri, token)`.

```javascript
const disposable = workspace.registerTextDocumentContentProvider('myscheme', {
  provideTextDocumentContent(uri) {
    return `# Content for ${uri.toString()}`
  }
})
context.subscriptions.push(disposable)
```

### jumpTo

```typescript
jumpTo(uri: string | URI, position?: Position | null, openCommand?: string): Promise<void>
```

Opens the given URI in the editor and optionally moves the cursor to `position`.

```javascript
await workspace.jumpTo('file:///path/to/file.ts', { line: 10, character: 0 })
```

### registerAutocmd

```typescript
registerAutocmd(autocmd: Autocmd, disposables?: Disposable[]): Disposable
```

Registers a Vim autocmd that invokes a JavaScript handler.

```javascript
const disposable = workspace.registerAutocmd({
  event: 'BufWritePost',
  pattern: '*.ts',
  callback: async () => {
    // handle write post
  }
})
context.subscriptions.push(disposable)
```

## Usage example

```javascript
const { workspace, Disposable } = require('coc.nvim')

async function activate(context) {
  // Listen for configuration changes
  context.subscriptions.push(
    workspace.onDidChangeConfiguration(e => {
      if (e.affectsConfiguration('myext')) {
        // reload settings
      }
    })
  )

  // Watch for document saves
  context.subscriptions.push(
    workspace.onDidSaveTextDocument(doc => {
      if (doc.languageId === 'typescript') {
        workspace.runCommand('tsc --noEmit')
      }
    })
  )

  // Read current configuration
  const config = workspace.getConfiguration('myext')
  const enabled = config.get<boolean>('enable', true)
}
```
