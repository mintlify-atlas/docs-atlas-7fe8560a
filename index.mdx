coc.nvim turns Vim and Neovim into a full-featured IDE by running a Node.js process that hosts language servers and VSCode-compatible extensions. It implements the Language Server Protocol (LSP 3.17) in full, giving you completion, diagnostics, code actions, hover docs, rename, semantic tokens, and more — all without slowing down your editor.

## Get started in minutes

<Steps>
  <Step title="Install Node.js and coc.nvim">
    Install Node.js >= 16.18.0, then add coc.nvim via your plugin manager (vim-plug, lazy.nvim, etc.)
  </Step>
  <Step title="Install a language extension">
    Run `:CocInstall coc-tsserver` (TypeScript), `:CocInstall coc-pyright` (Python), or configure a language server manually
  </Step>
  <Step title="Add key mappings">
    Add the recommended key mappings to your `init.vim` or `init.lua` to enable go-to-definition, hover, diagnostics navigation, and more
  </Step>
  <Step title="Open a project and code">
    Open any supported file — completion, diagnostics, and hover documentation are active immediately
  </Step>
</Steps>

## Why coc.nvim?

<CardGroup cols={2}>
  <Card title="Full LSP 3.17 support" icon="server">
    Every LSP feature including inline completion, inlay hints, semantic tokens, call hierarchy, and type hierarchy
  </Card>
  <Card title="VSCode extension ecosystem" icon="store">
    Install hundreds of community extensions via `:CocInstall` — the same model as VS Code extensions
  </Card>
  <Card title="Non-blocking by design" icon="bolt">
    All language intelligence runs in a separate Node.js process, keeping Vim/Neovim fast and responsive
  </Card>
  <Card title="Familiar configuration" icon="gear">
    Configure via `coc-settings.json` with JSON schema validation — the same format as VS Code's `settings.json`
  </Card>
</CardGroup>

<Note>
  coc.nvim requires **Vim >= 9.0.0438** or **Neovim >= 0.8.0**, and **Node.js >= 16.18.0**.
</Note>
